# front_middle_practice

개인 프론트엔드/미들웨어 역량 강화를 위해 운영하는 토이 프로젝트 저장소입니다.  
짧은 주기의 작은 실험을 반복하면서, **기획 -> 구현 -> 회고** 루프를 빠르게 돌리는 것을 목표로 합니다.

## 목표

- 프론트엔드 기본기(상태 관리, 비동기 처리, 라우팅, UI 구성) 반복 학습
- 실무에서 자주 만나는 문제를 작은 단위로 직접 구현
- 기술 선택의 이유를 기록하고 비교하는 습관 만들기
- 완성도보다 `빠른 시도 + 명확한 회고`에 집중

## 운영 방식

- 하나의 아이디어를 하나의 폴더(또는 브랜치)로 분리
- 작은 범위로 시작해서 기능을 점진적으로 확장
- 매 프로젝트마다 최소한 아래 항목 기록:
  - 무엇을 만들었는지
  - 어떤 문제를 만났는지
  - 다음에 개선할 점

## 폴더 구조 가이드

아래 구조를 기준으로 자유롭게 확장합니다.

```txt
front_middle_practice/
  projects/
    project-name-01/
    project-name-02/
  notes/
    retrospective.md
  README.md
```

- `projects/`: 실제 토이 프로젝트 코드
- `notes/`: 학습 메모, 트러블슈팅, 회고

## 새 토이 프로젝트 시작 템플릿

새 프로젝트를 만들 때 아래 체크리스트를 복붙해서 사용합니다.

```md
## [프로젝트 이름]
- 기간: YYYY-MM-DD ~ YYYY-MM-DD
- 주제: 
- 핵심 목표:
- 사용 기술:

### 구현 범위 (MVP)
- [ ] 기능 1
- [ ] 기능 2
- [ ] 기능 3

### 회고
- 잘한 점:
- 아쉬운 점:
- 다음 액션:
```

## 커밋/브랜치 간단 규칙

- 커밋은 가능한 작고 의미 단위로 쪼개기
- 메시지는 `무엇을 왜 바꿨는지` 중심으로 작성
- 큰 실험은 브랜치로 분리 후 정리해서 병합

## 아이디어 백로그

- [ ] 인증 흐름 연습 (로그인/로그아웃/보호 라우트)
- [ ] 검색 + 필터 + 정렬 UI
- [ ] 무한 스크롤/페이지네이션 비교 구현
- [ ] 폼 검증 및 에러 UX 개선
- [ ] 공통 컴포넌트/디자인 토큰 정리
